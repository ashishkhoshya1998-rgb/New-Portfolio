## Understanding the Problem Space

### Market Context

India's Goods and Services Tax (GST) regime requires over **1.4 crore businesses** to file GSTR-3B returns monthly or quarterly. These returns report sales, input tax credit (ITC) claims, tax liabilities, and refunds. The GSTN portal — the government's official platform — had several critical limitations:

- **Single-GSTIN Filing** — Businesses with multiple units (GSTINs) had to submit returns one at a time, which was time-consuming.
- **Manual Processes** — Users had to enter data manually from Excel sheets, increasing error and compliance risk.
- **Complex Reconciliation** — Comparing purchase registers (PR) with GSTR-2B (auto-generated from vendor filings) was complicated and prone to mistakes.

### Problem Statement

The GSTN portal and Clear GST 1.0 limited users to filing returns for only one GSTIN at a time, creating inefficiencies for businesses managing multiple GSTINs. Users struggled with data accuracy, reconciliation, and tax computation, making the process prone to errors and unnecessarily complex. Five core challenges:

<ProcessStep number="01" title="High Cognitive Load">
  Manual tracking and verification of data across GSTINs increased mental burden exponentially with scale.
</ProcessStep>

<ProcessStep number="02" title="Limited Automation">
  Restrictions on copy-pasting led to manual data entry errors — 82% of users copied from Excel.
</ProcessStep>

<ProcessStep number="03" title="Reconciliation Complexities">
  Matching GSTR-2B with purchase registers was cumbersome, affecting ITC claims — 65% of tickets were ITC mismatches.
</ProcessStep>

<ProcessStep number="04" title="Filing Flow Abandonment">
  Users frequently abandoned the process due to unclear data requirements and a standalone data source step.
</ProcessStep>

<ProcessStep number="05" title="Excel Dependency">
  Users were forced to rely on Excel for filtering, pivot tables, and bulk actions the platform should have provided.
</ProcessStep>

### User Research

To validate pain points, we conducted:

- **50+ Interviews** with MSMEs, large enterprises, and tax consultants
- **Usability Testing** on the existing GSTN portal and Clear GST 1.0 to observe friction points
- **Analysis of 500+ Customer Support Tickets** to identify recurring issues

## Design Process

### Design Goals

Based on the findings, I defined five core design goals:

<ProcessStep number="01" title="Enable Bulk Filing for Multiple GSTINs">
  **Why:** Reduce time spent on repetitive tasks for enterprises.<br/>
  **Approach:** Design centralised screens to manage all GSTINs in one place with batch actions.
</ProcessStep>

<ProcessStep number="02" title="Automate Reconciliation Between PR and GSTR-2B">
  **Why:** Minimise manual effort and errors in ITC claims.<br/>
  **Approach:** Build a pre-filing reconciliation engine with smart suggestions to take ITC actions.
</ProcessStep>

<ProcessStep number="03" title="Data Source Flexibility">
  **Why:** Let users choose datasets (PR, GSTR-2B) to auto-populate tables.<br/>
  **Approach:** Create a data source selection screen with table-wise mapping and recommendations.
</ProcessStep>

<ProcessStep number="04" title="Replicate Excel-Like Flexibility In-Platform">
  **Why:** Eliminate back-and-forth between Excel and the portal.<br/>
  **Approach:** Introduce filtering, sorting, and bulk actions in tables.
</ProcessStep>

<ProcessStep number="05" title="Generate Table-4 Directly from 2B vs PR Recon">
  **Why:** Simplify ITC computations and build user confidence.<br/>
  **Approach:** Auto-populate Table-4 based on reconciliation outcomes.
</ProcessStep>

### Designing the Solution

#### A. Data Source Selection Interface

**Purpose:** Assign datasets to populate each GSTR-3B table (e.g., PR for Table-4).

- **Table-Specific Mapping** — Dropdown menus for each table (e.g., "Table-4: GSTR-2B + PR")
- **Smart Defaults** — Auto-suggest optimal sources (e.g., GSTR-2B for SMEs)
- **Preview Mode** — Validate auto-filled values before proceeding

![Wireframe: Data Source Selection — table-wise mapping with smart defaults](/cs/gstr-wireframe-a.webp)

#### B. Multi-GSTIN Filing

**Purpose:** Centralise filing for businesses managing multiple units.

- **Batch Filing** — Select and submit returns for 10+ GSTINs in one click
- **Status Overview** — Track pending filings, tax liabilities, and ITC statuses
- **Drill-Down** — Access granular details (e.g., invoice-level data) per GSTIN

> "I need to see which GSTINs are due at a glance." — CFO, Manufacturing Firm

**Trade-off:** Simplified visual hierarchy over complex data density.

![Wireframe: Multi-GSTIN Dashboard — batch filing for 10+ GSTINs in one click](/cs/gstr-wireframe-b.webp)

#### C. Pre-Filing Reconciliation Engine

**Purpose:** Resolve discrepancies between GSTR-2B and PR for accurate Table-4.

- **Split-View Comparison** — Side-by-side display of GSTR-2B vs. PR with mismatches highlighted
- **Bulk Actions** — Approve/reject claims in bulk (e.g., "Accept All Matched Invoices")
- **Impact Preview** — Live updates to Table-4 as users reconcile

**Iteration:** Added vendor grouping and partial invoice matching post-launch.

![Wireframe: Reconciliation Engine — GSTR-2B vs PR split-view with bulk actions](/cs/gstr-wireframe-c.webp)

#### D. Table-4 Generation from Recon Actions

**Purpose:** Auto-populate Table-4 based on reconciliation decisions.

- **Real-Time Updates** — Show how ITC actions (claims, reversals) impact Table-4
- **Audit Trail** — Track changes with timestamps and user notes (e.g., "₹50,000 ITC claimed")
- **Inline Explanations** — Hover tooltips clarify how values are derived

> "I need to know why my ITC is blocked." — MSME Owner

![Wireframe: Table-4 auto-populated from reconciliation decisions — real-time audit trail](/cs/gstr-wireframe-d.webp)

### Information Architecture

I designed the Information Architecture with a clear focus on the objectives of each page. The entire workflow is divided into six key steps:

1. **GST Landing Page** — GSTR-3B Filing entry; access to other filings (GSTR-1, 6, 8, 9, 9C, ITC-04)
2. **Pre-Preparation (Data Source Setup)** — Select business unit (PAN or GSTINs), return period, activate GSTN connection, confirm & review data sources
3. **Data Preparation (Step 1/4)** — GSTIN activation, PR & SR import, 2B vs PR reconciliation for ITC (Table-4)
4. **Review Tax Calculation & Offset (Step 2/4)** — Detailed tax calculation, summary of liabilities, government vs Clear GST calculator, Rule 86B, create challans
5. **Upload to GSTN (Step 3/4)** — Upload data, post credit to ledger
6. **Filing & Status Review (Step 4/4)** — Select filing method (EVC/DSC), unified status dashboard

The IA was structured to balance compliance requirements with user efficiency. Each step addresses specific pain points while embedding design principles.

#### 1. Pre-Preparation Page (Data Source Setup)

**Purpose:** Configure data sources and activate GSTINs upfront.

- **Data Source Mapping** — Assign datasets (PR, GSTR-2B, Excel) to auto-populate tables
- **Smart Defaults** — Platform auto-suggested optimal sources (e.g., GSTR-2B for SMEs)
- **Bulk GSTIN Activation** — One-click connection to GSTN for all GSTINs under a PAN

**Design Rationale:**
- **Error Prevention** — Mandatory confirmation of data sources
- **Bulk Efficiency** — Enterprises activated 20+ GSTINs in under 10 minutes

#### 2. Data Preparation (Step 1/4)

**Purpose:** Import, validate, and reconcile data.

- **PR/Excel Import** — Drag-and-drop for purchase/sales registers
- **2B vs PR Reconciliation** — Side-by-side view with discrepancies flagged (e.g., "Invoice missing in GSTR-2B")
- **Bulk Actions** — Accept/reject ITC claims for 100+ invoices in one click

**Design Rationale:**
- **Excel-Like Flexibility** — Filter, sort data in-platform
- **Proactive Alerts** — Colour-coded warnings for mismatching data

![Data preparation — import, validate, and reconcile in one flow](/cs/gstr-data-prep.gif)

#### 3. Review Tax Calculation & Offset (Step 2/4)

**Purpose:** Finalise tax liability and credits.

- **Tax Calculator Toggle** — Compare government vs. Clear GST logic
- **Rule 86B Compliance** — Auto-flag cases where ITC exceeds 99% of liability

**Design Rationale:**
- **Transparency** — Side-by-side tax logic comparison built trust
- **Enterprise Scalability** — Saved 5+ hours for businesses with 100+ GSTINs

![Tax calculation review — government vs Clear GST logic, live liability comparison](/cs/gstr-tax-calc.gif)

#### 4. Upload to GSTN (Step 3/4)

**Purpose:** Submit data securely to the government portal.

- **OTP Activation Status** — Track authentication status for each GSTIN and enable quick login for streamlined uploading
- **Real-Time Sync** — Updated ledger balances post-upload
- **Multi-Challan Creation** — Generate challans for 100+ GSTINs simultaneously

**Design Rationale:**
- **Reduced Friction** — Users saved 10+ minutes per GSTIN
- **Audit Trail** — Logged timestamps for compliance tracking

![Upload to GSTN — OTP authentication and real-time ledger sync](/cs/gstr-upload.gif)

#### 5. Filing & Status Review (Step 4/4)

**Purpose:** Final submission and compliance tracking.

- **EVC/DSC Integration** — Pre-saved digital signatures for enterprises
- **Unified Status Dashboard** — Track filings across GSTINs in real-time

**Design Rationale:**
- **End-to-End Visibility** — Reduced follow-up queries by 40%
- **One-Click Filing** — Repeat users completed filings in under 30 seconds

![Filing and status dashboard — one-click filing, EVC/DSC integration](/cs/gstr-filing.gif)

## The 90-90-90 Drive

Post-GST 2.0 launch, we initiated a **customer adoption drive** led by Customer Success Managers (CSMs) and Product Managers (PMs). CSMs were incentivised to ensure users:

- **Adopted critical features** (e.g., data source selection, 2B vs PR reconciliation)
- **Filed GSTR-3B independently** without relying on CSMs

### Retrospective Insights

Monthly retros with CSMs revealed a critical flaw:

- **CSM Over-Reliance** — To meet targets, CSMs often performed tasks *for users* (configuring data sources, reconciling ITC, generating Table-4 for GSTR-3B).
- **Surface-Level Adoption** — Metrics looked positive, but users lacked *true understanding* of workflows.

> **Key Realisation:** *"Adoption ≠ Proficiency. Users were filing but not mastering the platform."*

## Pivot to User-Centric Research

To uncover real issues, we shifted focus from CSM-led handholding to **direct user engagement**.

### 1. Usability Testing with Real Users

**Method:** Shadowed users during GSTR-3B filing, asking open-ended questions:

- *"Where do you expect to find this detail?"*
- *"How would you solve this issue without the platform?"*

**Key Findings:**

**Data Source Page Confusion** — Users saw the standalone data source setup as a "gatekeeper" step, not a workflow enabler.

> *"Why can't I adjust sources directly in the GSTR-3B form?"* — MSME Owner

**Result:** 47% dropped off or called CSMs to bypass this step.

**Table-4 Reconciliation Friction** — Users struggled to map 2B vs PR reconciliation results to Table-4.

> *"I download the recon to Excel, create pivot tables, and cross-check. Your platform's logic is a black box."* — Tax Consultant

**Result:** 89% reverted to Excel, leading to mismatches and CSM escalations.

### 2. Funnel Analysis

**Tools:** Custom SQL Queries

**Key Drop-off Points Identified:**

- **Data Source Setup:** 47% abandonment due to unclear purpose
- **Reconciliation to Table-4:** 89% drop-off as users switched to Excel

### 3. Support Ticket Trends

**Top Issues:**

- *"How do I remove the data source page?"*
- *"Why does my Table-4 not match my Excel sheet?"*

**CSM Blind Spots:** CSMs themselves lacked clarity on Table-4 population logic, causing inconsistent guidance.

## Critical Issues Identified

<ProcessStep number="01" title="Data Source Page as a Barrier">
  Users wanted **contextual configuration** (adjust sources within the GSTR-3B form, not a separate step).<br/>
  *"Let me tweak sources while reviewing Table-4!"* — Enterprise User
</ProcessStep>

<ProcessStep number="02" title="Table-4 Opaqueness">
  Users needed **transparency** into how reconciliation actions (claiming ITC) impacted Table-4.<br/>
  *"If I reject an invoice, show me how it affects my final ITC!"* — SME Accountant
</ProcessStep>

<ProcessStep number="03" title="Excel Dependency">
  Lack of **Excel-like flexibility** (filtering, bulk actions) in-platform forced users to download data.
</ProcessStep>

## Iterative Solutions

### 1. Data Source Page Redesign

**Contextual Integration:** Moved source configuration **inside the GSTR-3B form** — eliminating the standalone "gatekeeper" step. Drop-off fell from 47% to near-zero.

![Iteration: Data source controls moved inline — from blocker to enabler](/cs/gstr-datasrc-redesign.gif)

### 2. Transparent Table-4 Workflow

**Live Impact Previews:** Introduced separate Table-4 generation to increase its visibility. The reconciliation table now displays real-time impact of each row on Table-4, increasing user confidence in data flow.

![Iteration: Table-4 live preview — every reconciliation action updates ITC in real time](/cs/gstr-table4.gif)

### 3. Excel-Free Features

**Intuitive data grouping that eliminates the need for pivot tables:** Introduced grouping and subgrouping capabilities to enable efficient and accurate bulk actions.

![Iteration: Native grouping eliminates Excel dependency](/cs/gstr-excel-free.gif)

### 4. Multi-GSTIN Auto Activation

**Instant GSTIN Activation with Auto-OTP Capture:** Streamlined the filing process with automated OTP capture for multiple GSTINs (with user consent), significantly reducing processing time.

## Key Learnings

**1. Adoption ≠ Proficiency.** Incentivising usage without ensuring understanding backfires.

**2. Context > Isolation.** Integrate critical steps (like data sources) into natural workflows.

**3. Transparency Builds Trust.** Users need to *see* how actions impact outcomes.
